# Семестровая работа: Распознавание рукописных математических конспектов и перевод в LaTeX

## Описание проекта

Целью проекта является разработка нейронной сети и приложения,
которое принимает фотографии страниц рукописного математического конспекта
**на русском языке** (текст + формулы) и генерирует готовый LaTeX-документ.

Модель должна распознавать:

- **Русский рукописный текст** — предложения, определения, теоремы
  (кириллица, знаки препинания, нумерация)
- **Рукописные математические формулы** — интегралы, дроби, суммы,
  матрицы, греческие буквы и прочая LaTeX-разметка
- **Смешанные строки** — текст с вкраплениями inline-формул
  (например: «Пусть $f(x)$ — непрерывная функция на $[a, b]$»)

Студент самостоятельно проектирует и реализует полный pipeline обработки:
предобработку изображений, автоматическую нарезку страниц на строки,
токенизацию LaTeX-разметки, архитектуру нейронной сети (энкодер и декодер),
мульти-датасетное обучение, стратегию декодирования
и приложение с веб-интерфейсом.

> Модель должна быть написана и обучена **полностью с нуля** на фреймворке
> PyTorch. Использование любых готовых предобученных моделей запрещено.


## Конфигурационная система

Все параметры проекта (архитектура модели, обучение, данные, пути)
выносятся в **единый конфигурационный модуль** (`config.py`).

Требования:

- **GPU-профили** — предустановленные наборы параметров для разных видеокарт
  (объём VRAM определяет batch size, размерность модели, число слоёв и т.д.).
  Переключение между профилями — одна строка кода или аргумент командной строки.
- **Градиент-аккумуляция** — эффективный batch size не зависит от VRAM:
  маленький физический батч компенсируется накоплением градиентов.
- **Легко расширяемая структура** — добавление нового GPU-профиля
  или нового набора данных не требует изменения кода обучения.


## Требования к архитектуре нейронной сети

### Энкодер (Hybrid CNN + ViT)

Для извлечения визуальных признаков из изображений произвольной ширины
используется гибридная архитектура **CNN-бекбон + Vision Transformer (ViT)**.

Студент реализует:

- **Асимметричный CNN Downsampling** — свёрточная сеть, которая агрессивно
  сжимает высоту изображения, но бережно сжимает ширину, превращая длинный
  мелкий почерк в 1D-последовательность признаков без потери деталей.
  Степень сжатия по каждой оси — конфигурируемый параметр.
- **Positional Encoding** — математические (Sine-Cosine) или относительные
  (RoPE) пространственные эмбеддинги, позволяющие принимать изображения
  **любой** неограниченной ширины на этапе инференса.
- **Transformer Encoder** — стек слоёв `nn.TransformerEncoderLayer`
  с Multi-Head Self-Attention и Feed-Forward подсетью.
  Количество слоёв, размерность и число голов — конфигурируемые параметры.
- **Layer Normalization** — на выходе энкодера.

### Декодер (Transformer Decoder)

Для авторегрессивной генерации последовательности LaTeX-токенов
используется **Transformer Decoder**.

Студент реализует:

- **Token Embedding** — `nn.Embedding` для представления входных токенов.
  Словарь покрывает LaTeX-команды, кириллицу, латиницу, цифры,
  математические символы и спецтокены.
- **Positional Encoding** — обучаемые позиционные эмбеддинги
  с интерполяцией для поддержки последовательностей произвольной длины.
- **Transformer Decoder** — стек слоёв на основе Multi-Head Attention.
  Использование `F.scaled_dot_product_attention` (FlashAttention)
  для поддержания длинного контекста при ограниченной памяти.
  Количество слоёв, размерность и число голов — конфигурируемые параметры.
- **Output Projection** — `nn.Linear(d_model, vocab_size)` для получения
  логитов по словарю.
- **Layer Normalization**

### Полная модель

Класс `Notes2LaTeX(nn.Module)` объединяет энкодер и декодер:

1. Изображение проходит через ViT Encoder → `memory`
2. Создаётся causal mask для целевой последовательности
3. `memory` и целевые токены подаются в декодер
4. На выходе — логиты размерности `[B, seq_len, vocab_size]`

Функция `count_parameters(model)` выводит точное число обучаемых параметров
после сборки модели.


## Требования к данным

### Мульти-датасетный подход

Для обучения модели распознаванию **и текста, и формул** на русском языке
необходимо использовать **несколько источников данных** одновременно:

- **Открытый датасет формул** (например, im2latex-100k) — обучает модель
  генерировать LaTeX-разметку математических выражений.
- **Датасет рукописной математики** (например, CROHME) — обучает модель
  распознавать рукописные (а не печатные) формулы.
- **Синтетический датасет** — автоматически сгенерированные пары
  «изображение строки → LaTeX», содержащие русский текст, формулы
  и смешанные строки. Основной источник данных для обучения распознаванию
  кириллицы в контексте математических конспектов.
- **Собственный датасет** — рукописные конспекты студента, размеченные
  вручную. Используется для fine-tuning на конкретный почерк.

Конфигурация определяет, какие датасеты участвуют в обучении
и с какими весами они смешиваются.

### Синтетический датасет

Ключевой компонент для обучения распознаванию русского текста с формулами.
Pipeline генерации:

1. **Сбор LaTeX-шаблонов** — фрагменты реальных конспектов, учебников,
   задачников (русский текст + формулы, чистые формулы, чистый текст)
2. **Рендеринг** — компиляция LaTeX → PDF → растеризация в изображения
3. **Нарезка на строки** — получение отдельных строк с точным ground truth
4. **Аугментации «под почерк»** — морфологические искажения, вариация
   толщины линий, эластичные деформации, шум, размытие
5. **Сохранение** — пары (изображение строки, LaTeX-код)

Объём синтетического датасета, набор шрифтов и параметры аугментаций —
конфигурируемые параметры.

### Собственный датасет

Для дообучения (fine-tuning) модели на конкретный почерк студент
собирает собственный набор данных из рукописных конспектов
достаточного объёма.

Инструментарий сбора:

- **Нарезка страниц на строки** — алгоритмы Computer Vision
  (бинаризация, компенсация наклона, проекционный анализ)
  с GUI для ручной коррекции линий разреза
- **Авторазметка** — использование внешнего LLM API для предложения
  LaTeX-кода по изображению строки (черновая разметка)
- **GUI-верификация** — приложение для ручной проверки и правки
  предложенной разметки, с сохранением прогресса

### Унифицированный токенизатор LaTeX

Студент реализует класс `LaTeXTokenizer` с Regex-токенизацией,
поддерживающий **полный спектр символов** конспекта:

- LaTeX-команды (`\frac`, `\int`, `\text` и т.д.) — один токен
- Кириллические символы (а–я, А–Я, ё, Ё) — один токен на букву
- Латинские символы, цифры — один токен
- Математические символы и скобки — один токен
- Пробел — отдельный токен (критично для русского текста)

Методы:

- `tokenize(formula)` — разбиение LaTeX на токены
- `build_vocab(formulas)` — построение словаря из всех датасетов
- `encode(formula, max_len)` — формула → список индексов с паддингом
- `decode(ids)` — список индексов → LaTeX-строка

Специальные токены:

| Токен | Назначение |
|---|---|
| `<PAD>` | Паддинг до одинаковой длины в батче |
| `<SOS>` | Начало последовательности |
| `<EOS>` | Конец последовательности |
| `<UNK>` | Неизвестный токен |

Проверка: `decode(encode(formula))` должен возвращать исходную формулу.

### Предобработка изображений

Функция `preprocess_image(image_path)` реализует полный pipeline:

1. Загрузка изображения в grayscale
2. Data Augmentation (морфологические искажения,
   эмулирующие разную толщину чернил, Гауссов шум и расфокус камеры) —
   применяется только при обучении
3. Адаптивная бинаризация
4. Обрезка (Crop) по контенту
5. Resize до фиксированной высоты **с сохранением пропорций ширины**
6. Нормализация в тензор PyTorch

Выходной формат: `torch.Tensor` shape `[1, H, W]` с динамической шириной.

### Загрузка данных

Загрузчик данных реализует:

- **Мульти-датасетный DataLoader** — объединение нескольких датасетов
  в один поток с конфигурируемыми весами выборки
- **Динамический паддинг** — паддинг формул и изображений осуществляется
  исключительно до максимума в текущем батче, а не статически
- **Адаптивное батчирование (BucketBatchSampler)** — группировка формул
  одинаковой длины в батчи и уменьшение размера батча
  для сверхдлинных строчек, предотвращая Out-Of-Memory


## Требования к обучению

### Многоэтапное обучение

Обучение модели проводится в несколько этапов:

1. **Pre-training на формулах** — модель учится базовой генерации LaTeX
   по изображениям печатных формул (открытый датасет)
2. **Смешанное обучение** — добавляются синтетические данные
   (русский текст + формулы) и опционально другие датасеты.
   Модель учится распознавать кириллицу и смешанные строки.
   Применяется Curriculum Learning (от простых к сложным примерам).
3. **Fine-tuning** — дообучение на собственном рукописном датасете
   со сниженным learning rate и early stopping.
   Сравнение метрик до и после fine-tuning.

### Оптимизация производительности

- **Mixed Precision (AMP)** — вычисления в смешанной точности,
  экономящие видеопамять и ускоряющие обучение
- **Gradient Accumulation** — эмуляция большого батча
  при ограниченном VRAM
- **Gradient Checkpointing** — опциональная экономия VRAM
  за счёт перевычисления активаций (для сверхбольших моделей)
- **FlashAttention** через `F.scaled_dot_product_attention`
- **Tensor Cores** — `torch.set_float32_matmul_precision('high')`
  для ускорения на NVIDIA RTX
- **Параллельная загрузка данных** — `num_workers` + `prefetch_factor`

### Поиск гиперпараметров

Скрипт `tune.py` реализует поиск гиперпараметров (Bayesian Search)
через Optuna. Оптимизируемые параметры и их границы задаются
в конфигурации. Лучшая конфигурация сохраняется для финального обучения.

### Стратегия декодирования (инференс)

Функция `beam_search(model, image, tokenizer, ...)`:

1. Прогнать изображение через энкодер → `memory`
2. Инициализация: `[<SOS>]`
3. На каждом шаге — расширение top-N гипотез
4. Остановка при `<EOS>` или достижении максимальной длины
5. Возврат лучшей гипотезы (по `score / length`)

Параметры beam search (beam_size, max_len, length penalty)
задаются в конфигурации.


## Требования к метрикам оценки качества

Скрипт `evaluate.py` загружает модель из checkpoint, прогоняет
тестовый набор с Beam Search и вычисляет следующие метрики:

| Метрика | Описание |
|---|---|
| **BLEU-4** | Близость сгенерированного кода к эталону |
| **ExactMatch** | Доля формул / строк, распознанных абсолютно точно |
| **Token Accuracy** | Точность по отдельным токенам (без учёта `<PAD>`) |
| **CER** | Character Error Rate — посимвольная ошибка (для текста) |
| **Edit Distance** | Расстояние Левенштейна (нормализованное) |

Целевые значения метрик определяются в плане реализации.

Скрипт выводит таблицу результатов и сохраняет примеры предсказаний.


## Требования к приложению

Финальное приложение реализуется в архитектуре
**FastAPI** (бэкенд) + **Streamlit** (фронтенд).

### Бэкенд (FastAPI)

REST API для распознавания рукописных конспектов:

- `POST /predict` — принимает изображение одной строки, возвращает LaTeX-код
- `POST /predict-page` — принимает изображение страницы конспекта,
  автоматически нарезает на строки, распознаёт каждую строку
  и возвращает собранный LaTeX-документ

API загружает модель из checkpoint при старте и обслуживает запросы на инференс.

### Фронтенд (Streamlit)

Веб-интерфейс для пользователя:

- Загрузка одной или нескольких фотографий страниц конспекта (JPG/PNG)
- Автоматическая отправка на бэкенд для распознавания
- Отображение результата: LaTeX-код в текстовом поле с копированием
  и рендер формул для визуальной проверки
- Возможность скачать итоговый `.tex` файл

### Pipeline обработки страницы

```
Фото страницы → Нарезка на строки → Распознавание каждой строки → Сборка LaTeX-документа
```

Нарезка страницы на строки переиспользует алгоритмы Computer Vision
(бинаризация, компенсация наклона, поиск строк), разработанные
в процессе сбора собственного датасета.

## Требования к обработке ошибок

- Предобработка: корректная обработка повреждённых/отсутствующих изображений
- Токенизатор: обработка неизвестных символов через `<UNK>`
- Обучение: graceful shutdown, сохранение последнего checkpoint при прерывании
- Инференс: валидация входного изображения (формат, размер)
- API: корректные HTTP-ответы с описанием ошибок
- Синтетический генератор: пропуск шаблонов с ошибками компиляции


## Критерии оценки

Работа считается выполненной, если удовлетворены следующие критерии:

- Модель написана и обучена полностью с нуля на PyTorch (без предобученных моделей)
- Реализован полный pipeline: предобработка → токенизация → обучение → инференс
- Модель обучена на нескольких датасетах (формулы + синтетический русский текст)
- Модель распознаёт русский рукописный текст, математические формулы и смешанные строки
- Собран и размечен собственный датасет из рукописных конспектов
- Проведён fine-tuning на собственном датасете
- Достигнуты целевые значения метрик (определены в плане реализации)
- Реализован Beam Search декодирование
- Приложение принимает фотографии страниц конспектов, автоматически нарезает на строки и выдаёт готовый LaTeX-документ
- Реализована клиент-серверная архитектура: FastAPI (бэкенд) + Streamlit (фронтенд)
- Все параметры конфигурируемы через единый модуль с поддержкой GPU-профилей
- Код структурирован по модулям, читаем и документирован
- Код размещён в Git-репозитории с историей коммитов
- `README.md` содержит описание, инструкции и результаты

### Дополнительные требования (опционально)

- Greedy decoding как альтернатива Beam Search для сравнения
- Экспорт модели в ONNX для ускорения инференса
- Развёртывание приложения на удалённом сервере
- Gradient Checkpointing для экономии VRAM
