# Семестровая работа: Распознавание рукописных математических конспектов и перевод в LaTeX

## Описание проекта

Целью проекта является разработка нейронной сети и приложения,
которое принимает фотографии страниц рукописного математического конспекта
и генерирует готовый LaTeX-документ.

Студент самостоятельно проектирует и реализует полный pipeline обработки:
предобработку изображений, автоматическую нарезку страниц на строки,
токенизацию LaTeX-разметки, архитектуру нейронной сети (энкодер и декодер),
цикл обучения, стратегию декодирования и приложение с веб-интерфейсом.

> Модель должна быть написана и обучена **полностью с нуля** на фреймворке
> PyTorch. Использование любых готовых предобученных моделей запрещено.


## Требования к архитектуре нейронной сети

### Энкодер (Hybrid CNN + ViT)

Для извлечения визуальных признаков из изображений произвольной ширины используется гибридная архитектура **CNN-бекбон + Vision Transformer (ViT)**.

Студент реализует:

- **Асимметричный CNN Downsampling** — свёрточная сеть, которая агрессивно сжимает высоту изображения (например, в 32 раза), но бережно сжимает ширину (например, в 4-8 раз), превращая длинный мелкий почерк в 1D/2D последовательность признаков без потери деталей.
- **Positional Encoding** — математические (Sine-Cosine) или относительные (RoPE) пространственные эмбеддинги, позволяющие принимать изображения **любой** неограниченной ширины на этапе инференса.
- **Transformer Encoder** — стек слоёв `nn.TransformerEncoderLayer`.
  с Multi-Head Self-Attention и Feed-Forward подсетью
- **Layer Normalization** — на выходе энкодера

Параметры по умолчанию:

| Параметр | Значение |
|---|---|
| Размер входа | `[B, 1, 128, W]` (динамическая ширина W) |
| CNN Downsample | Высота -> 1x, Ширина -> W/16 |
| `d_model` | `256` |
| Число голов внимания | `8` |
| Число слоёв энкодера | `4 - 6` |
| Выход энкодера | `[B, W/16, 256]` |

### Декодер

Для авторегрессивной генерации последовательности LaTeX-токенов
используется **Transformer Decoder**.

Студент реализует:

- **Token Embedding** — `nn.Embedding(vocab_size, d_model, padding_idx=0)`
  для представления входных токенов
- **Positional Encoding** — обучаемые позиционные эмбеддинги
  (`nn.Embedding(max_len, d_model)`) или относительные алгоритмы.
- **Transformer Decoder** — стек слоёв на основе Multi-Head Attention.
  > **Важно:** использование `F.scaled_dot_product_attention` (FlashAttention) для поддержания контекста длиной 1000+ токенов при ограниченной памяти (8GB VRAM).
- **Output Projection** — `nn.Linear(d_model, vocab_size)` для получения
  логитов по словарю
- **Layer Normalization**

Параметры по умолчанию:

| Параметр | Значение |
|---|---|
| `d_model` | `256` |
| Число голов внимания (`nhead`) | `8` |
| Число слоёв декодера | `4 - 6` |
| Максимальная длина последовательности | `500 - 1000` токенов |

### Полная модель

Класс `Notes2LaTeX(nn.Module)` объединяет энкодер и декодер:

1. Изображение проходит через ViT Encoder → `memory`
2. Создаётся causal mask для целевой последовательности
3. `memory` и целевые токены подаются в декодер
4. На выходе — логиты размерности `[B, seq_len, vocab_size]`

Функция `count_parameters(model)` выводит точное число обучаемых параметров
после сборки модели.

## Требования к данным

### Базовый датасет

Для обучения модели с нуля используется открытый датасет пар
«изображение формулы → LaTeX-код» (например, `im2latex-100k` или аналогичный).

Конкретный датасет и его формат определяются на этапе реализации.

Студент реализует загрузчик данных (`Dataset`, `DataLoader`) с использованием:
- **Динамического паддинга** — паддинг формул и изображений осуществляется
  исключительно до максимума в текущем батче, а не статически, экономя память.
- **Адаптивного батчирования (BucketBatchSampler)** — группировка формул 
  одинаковой длины в батчи и уменьшение размера батча `batch_size` 
  для сверхдлинных строчек (> 500 токенов), предотвращая Out-Of-Memory.

### Собственный датасет

Для дообучения (fine-tuning) модели на конкретный почерк студент
собирает собственный набор данных из рукописных конспектов
(не менее 150 размеченных строк).

Способ нарезки страниц на строки, метод разметки и инструменты
верификации определяются на этапе реализации.

### Токенизатор LaTeX

Студент реализует класс `LaTeXTokenizer` с Regex-токенизацией:

- `tokenize(formula)` — разбиение LaTeX на токены
  (команды `\frac`, `\int` = 1 токен; символы и скобки = 1 токен)
- `build_vocab(formulas)` — построение словаря из всего датасета
- `encode(formula, max_len)` — формула → список индексов с паддингом
- `decode(ids)` — список индексов → LaTeX-строка

Специальные токены:

| Токен | Индекс | Назначение |
|---|---|---|
| `<PAD>` | `0` | Паддинг до одинаковой длины в батче |
| `<SOS>` | `1` | Начало последовательности |
| `<EOS>` | `2` | Конец последовательности |
| `<UNK>` | `3` | Неизвестный токен |

Проверка: `decode(encode(formula))` должен возвращать исходную формулу.

### Предобработка изображений

Функция `preprocess_image(image_path)` реализует полный pipeline:

1. Загрузка изображения в grayscale
2. Предварительная Data Augmentation (морфологические искажения, 
   эмулирующие разную толщину чернил, Гауссов шум матрицы и расфокус камеры)
3. Адаптивная бинаризация 
4. Обрезка (Crop) по контенту
5. Resize до фиксированной высоты `128` пикселей **с сохранением пропорций ширины**
6. Нормализация в тензор PyTorch (`min=-1.0`, `max=1.0`)

Выходной формат: `torch.Tensor` shape `[1, 128, W]`.

## Требования к обучению

### Базовое обучение

Скрипт `train.py` реализует цикл обучения:

1. Загрузка датасета и токенизатора
2. Создание модели `Notes2LaTeX`
3. Оптимизатор, функция потерь и планировщик learning rate
   (рекомендуемые: `AdamW`, `CrossEntropyLoss` с `ignore_index=0`, `CosineAnnealingLR`)
4. Цикл по эпохам:
   - `train_epoch()` → train loss
   - `validate()` → val loss, val BLEU
   - Сохранение checkpoint при улучшении val loss
5. Построение графиков обучения

### Fine-tuning (собственный датасет)

Скрипт `finetune.py`:

- Загрузка лучшего checkpoint из базового обучения
- Создание Dataset из собственного датасета
- Дообучение со сниженным learning rate
- Early stopping
- Сравнение метрик до и после fine-tuning

### Advanced Оптимизация Обучения (MLOps & Performance)

Чтобы выжать **максимальную производительность и качество**, мы внедряем следующие Engineering/MLOps практики:

1. **Hyperparameter Tuning (Optuna)**
Скрипт `tune.py` будет разворачивать поиск гиперпараметров (Grid/Bayesian Search). Он научит модель находить идеальный баланс `learning_rate`, `n_heads`, `d_model` и параметров аугментации, сохраняя лучшую конфигурацию для финального обучения.

2. **Двухступенчатая обработка датасета (Pre-Cache + RAM Streaming)**
- **Cold Preprocessing:** Мы напишем скрипт-подготовщик датасета. Он *один раз* пройдется по всем 100 000 картинкам, обрежет их (Crop), удалит пустые поля и сожмет в оптимизированный формат (например, LMDB или HDF5). Загрузка с SSD ускорится в десятки раз.
- **Hot Augmentation (RAM):** Перезаписывать датасет на диск между эпохами мы **не будем**, так как запись 100к искаженных картинок каждый раз упрется в пропускную способность SSD (5-7 ГБ/сек) и будет сжигать его ресурс. Вместо этого мы включим многопроцессорность (`num_workers=8` и `prefetch_factor`). Процессор будет генерировать случайные изгибы ElasticTransform прямо в сверхбыстрой оперативной памяти (DDR5/DDR4 со скоростью 50+ ГБ/сек) и отправлять напрямую в видеокарту. Это даст предельную скорость.

3. **PyTorch Max Performance (Без `torch.compile`, ввиду ограничений Windows)**
- `torch.set_float32_matmul_precision('high')` — ускорение через Tensor Cores на NVIDIA RTX.
- `torch.amp.autocast` — вычисления в смешанной точности (FP16/BF16), что экономит до 50% видеопамяти и пропорционально ускоряет подсчеты.
- **FlashAttention-2** и **Fused Optimizers** (объединение математики оптимизатора).

### Стратегия декодирования (инференс)

Функция `beam_search(model, image, tokenizer, beam_size=5, max_len=150)`:

1. Прогнать изображение через энкодер → `memory`
2. Инициализация: `[<SOS>]`
3. На каждом шаге — расширение top-N гипотез
4. Остановка при `<EOS>` или достижении `max_len`
5. Возврат лучшей гипотезы (по `score / length`)

## Требования к метрикам оценки качества

Скрипт `evaluate.py` загружает модель из checkpoint, прогоняет
тестовый набор с Beam Search и вычисляет следующие метрики:

| Метрика | Описание | Целевое значение |
|---|---|---|
| **BLEU-4** | Близость сгенерированного кода к эталону | > 0.70 на собственном датасете |
| **ExactMatch** | Доля формул, распознанных абсолютно точно | ≥ 45% |
| **Token Accuracy** | Точность по отдельным токенам (без учёта `<PAD>`) | — |

Скрипт выводит таблицу результатов и сохраняет примеры предсказаний.

## Требования к приложению

Финальное приложение реализуется в архитектуре **FastAPI** (бэкенд) + **Streamlit** (фронтенд).

### Бэкенд (FastAPI)

REST API для распознавания рукописных конспектов:

- `POST /predict` — принимает изображение одной строки, возвращает LaTeX-код
- `POST /predict-page` — принимает изображение страницы конспекта,
  автоматически нарезает на строки, распознаёт каждую строку
  и возвращает собранный LaTeX-документ

API загружает модель из checkpoint при старте и обслуживает запросы на инференс.

### Фронтенд (Streamlit)

Веб-интерфейс для пользователя:

- Загрузка одной или нескольких фотографий страниц конспекта (JPG/PNG)
- Автоматическая отправка на бэкенд для распознавания
- Отображение результата: LaTeX-код в текстовом поле с копированием
  и рендер формул для визуальной проверки
- Возможность скачать итоговый `.tex` файл

### Pipeline обработки страницы

```
Фото страницы → Нарезка на строки → Распознавание каждой строки → Сборка LaTeX-документа
```

Нарезка страницы на строки переиспользует алгоритмы Computer Vision
(бинаризация, компенсация наклона, поиск строк), разработанные
в процессе сбора собственного датасета.

- Предобработка: корректная обработка повреждённых/отсутствующих изображений
